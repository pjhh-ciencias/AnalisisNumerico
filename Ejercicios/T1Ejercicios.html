Ejercicio

Demuestra que si $f \in C[a,b]$ y $f(a) f(b) < 0 $, entonces existe $c \in (a,b)$ tal que $f(c)=0.$

Ejercicio

Construye un programa que implemente el Método de Horner para evaluar polinomios. El programa debe solicitar el grado $n$ del polinomio $p(x)$, el punto $c$ en el que se debe evaluar el polinomio y regresar $p(c)$.

Ejercicio

Usa el programa creado en el Ejercicio 1.2 para hallar $p(c)$ en los siguientes casos:

  1. $p(x)= x^4 + x^3 -13x^2 -x -12, c=3$
  2. $p(x)= 2x^7 + x^6 + x^5 -2^4 -x +23, c=-1$

Ejercicio

Construye un programa que reciba un número entero positivo $N$ y devuelva la representación binaria de $N$.

Ejercicio

Construye un programa que reciba un archivo .txt con la representación binaria de un número entero positivo y devuelva su forma decimal (base $10$) de los siguientes números:

  1. $10101_{\text{dos}}$
  2. $11111110_{\text{dos}}$
  3. $111000_{\text{dos}}$
  4. $1000000111_{\text{dos}}$

Ejercicio

Construye un programa que reciba un archivo .txt con la representación de una fracción binaria y devuelva su forma decimal (base $10$) de los siguientes números:

  1. $11011_{\text{dos}}$
  2. $1010101_{\text{dos}}$
  3. $10101_{\text{dos}}$
  4. $110110110_{\text{dos}}$

Ejercicio

Construye un programa que reciba un fracción decimal $0< R < 1$ y devuelva la correspondiente fracción binaria de $R$, represente unicamente $32$ bits, i.e. $32$ digitos despues del punto.

Ejercicio

Construye un programa que reciba un archivo .txt con la representación binaria de un número decimal positivo entre $0$ y $1$ (i.e. recibirá una fracción binaria), y devuelva su forma decimal (base $10$) de los siguientes números:

  1. $0.10101_{\text{dos}}$
  2. $0.11111110_{\text{dos}}$
  3. $0.111000_{\text{dos}}$
  4. $0.1000000111_{\text{dos}}$

Ejercicio

Construye un programa que reciba un número real positivo $x$, y devuelva la correspondiente representación binaria de $x$, use $32$ bits para representar la parte entera y $32$ bits para representar la parte decimal. Por ejemplo, si $x=1.5$ el programa devolvera $$00000000000000000000000000000001.10000000000000000000000000000000$$

Sugerencia: Convierta la parte entera y la parte fraccionaria por separado.
